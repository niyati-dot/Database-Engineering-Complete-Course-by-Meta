The following reference sheet outlines some common MySQL syntax that you can use when working with virtual tables in MySQL.

Creating virtual tables in MySQL

A view in MySQL is created with the CREATE VIEW statement.

Syntax: 

CREATE VIEW TableView AS SELECT STATEMENT;

Example: 

 CREATE VIEW OrdersView AS SELECT OrderID, Cost FROM Orders;

This creates a virtual table that enables users to show the Order ID and Cost columns from the Orders table.

Retrieving Data from Virtual Tables

You can retrieve data from the virtual table in the same way you retrieve data from the normal table.

Syntax: 

SELECT * FROM TableView; 

Example: 

SELECT * FROM OrdersView; 

Updating Data in Virtual Tables

The virtual table itself does not store data physically, however, it is possible to update the base table data using the virtual table in the same way you update data in a normal table.

Syntax: 

Update TableView set ColumnName = value where condition;

Example: 

Update OrdersView set Cost = 500 WHERE OrderID = 1;

This statement updates the base Orders table via the OrdersView table.

Renaming Virtual Tables

In case you want to rename the virtual table, you can do that exactly in the same way you rename a normal table.

Syntax: 

RENAME TABLE VirtualTableName TO NewVirtualTableName; 

Example: 

RENAME TABLE OrdersView TO OrdersViewTable; 

This changes the OrdersView table name to a new name “OrdersViewTable”.

Dropping Virtual Tables

You can drop the virtual table using the SQL DROP VIEW command

Syntax: 

DROP VIEW VirtualTableName; 

Example: 

DROP VIEW OrdersViewTable; 

This will delete the virtual table view without impacting the actual Orders table.
