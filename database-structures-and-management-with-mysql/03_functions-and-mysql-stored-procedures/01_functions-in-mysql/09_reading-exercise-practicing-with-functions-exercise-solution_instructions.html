Task 1 solution: Use the MySQL CEIL function in order to express the cost after the discount in the form of the smallest integer.

H​ere's the original result that you get without using the CEIL function.

Here’s the solution with the help of the MySQL CEIL function. 

SELECT ClientID, OrderID, CEIL((Cost -(Cost * 5 /100))) AS afterDiscount 
FROM client_orders
WHERE ItemID = 4;

This is the output result.

Pay attention to the afterDiscount column. For example, you can see that the value 1187.5000 in the original result is now converted to a value of 1188, which is the smallest integer that’s not less than 1187.5000.

Task 2 solution: Format the afterDiscount column value from the earlier result for 5% discount in '#,###,###.##' format rounded to 2 decimal places using the FORMAT function. 

SELECT ClientID, OrderID, FORMAT ((Cost -(Cost * 5 /100)), 2) AS afterDiscount 
FROM client_orders  
WHERE ItemID = 4;

Notice that the value to be formatted is given as: Cost -(Cost * 5 /100)), which is the after-discount value, and decimal places are given as 2.

Task 3 solution: Find the expected delivery dates for their orders. The scheduled delivery date is 30 days after the order date. Use the ADDDATE function.

Here’s the query using the 1st form of ADDDATE to find the expected delivery dates for the orders that M & G have placed. The expected delivery date is 30 days after the order date. 

SELECT ADDDATE(OrderDate,INTERVAL 30 DAY) AS ExpectedDelDate 
FROM mg_orders;

Here’s the query using the second form: 

SELECT ADDDATE(OrderDate,30) AS ExpectedDelDate FROM mg_orders;

Task 4 solution : Generate data required for a report with details of all orders that have not yet been delivered. The DeliveryDate column has a NULL value for orders not yet delivered. It would help if you showed a value of 'NOT DELIVERED' instead of showing NULL for orders that are not yet delivered. Use the COALESCE function to do this. 

SELECT OrderID, ItemID, Quantity, Cost,  OrderDate,COALESCE (DeliveryDate,'NOT DELIVERED'), OrderStatus 
FROM mg_orders;

The result is as follows.

Here, if the value in the DeliveryDate column is NULL, the COALESCE function will substitute it with the 'NOT DELIVERED' string. Otherwise, it returns the value of the DeliveryDate column.

Task 5 solution : G​enerate data required for the report by retrieving a list of M&G orders that are yet to be delivered. These are the orders with a status of 'In Progress', using the NULLIF function. This is the query that gives the desired result, using the COALESCE function.

Here’s how you can retrieve a list of M & G orders that need following up. In other words, orders that have a status of ‘In Progress’.

The query can be written using the NULLIF function: 

SELECT OrderID, NULLIF(OrderStatus,'In Progress') AS status 
FROM mg_orders;

This is the result:

Here, the NULLIF function checks if the order status is equal to ‘In Progress’. If it is, it returns NULL. If not, it returns the value of the order status. Orders that return a NULL value must be followed up, as they are still in progress.
